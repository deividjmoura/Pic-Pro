Certo! Como vai ser:


Sempre que formos trabalhar iremos antes de tudo dar um git pull (sempre
observando se está no repositório certo ;) pra que?!

o pull irá atualizar nosso repositório local para a ultima versão.


Feito isso iremos usar o git checkout -b (branch) - onde está
(branch) colocamos o nome da branch, SEM colchetes. EXEMPLO:
git checkout -b nova_branch


Isso criará uma cópia da -main onde podemos trabalhar sem
estragar o codigo que está certo. SEMPRE CRIAR NOVA BRANCH quando for
trabalhar, pois está será revisionada por todos os integrantes antes do -merge
que é colocar as alterções revisionadas no arquivo principal.


PS. Geralmene quando damos o comando para criar nova branch,
automaticamente vamos para a mesma.
git checkout -b nova_branch iria me jogar direto na nova_branch, mas
como é melhor prevenir do que remediar, usem git branch para ver em
qual branch está, a navegação entre branchs é feita através do
git checkout nome_da_branch_aqui.
